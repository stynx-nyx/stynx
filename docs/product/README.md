# Product

**Authority:** Owner (Constitution Article 6).

Stynx is a framework repository, so this substrate does not define a single
tenant application's business backlog. Product-facing examples live here only
when they describe reference journeys, adoption scenarios, or framework
capabilities that consumers can reuse.

## Current Product Artifacts

- [use-cases/](use-cases/) — reference-app use cases used by coverage and RBAC
  diagnostics.
- [draft/blueprints/](draft/blueprints/) — DEVAI module blueprints and scaffold
  inputs used during the C-4 pilot.

Business rules for a consuming application should live in that application's
own `docs/product/` substrate, not in stynx.
