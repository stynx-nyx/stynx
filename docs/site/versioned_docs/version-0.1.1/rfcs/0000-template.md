# RFC 0000: Template

- Status: Template
- Date: YYYY-MM-DD

## Problem

Describe the invariant, architecture, tenancy, permissions, AWS dependency, or core-major change.

## Constraints

List security, migration, compatibility, operational, and compliance constraints.

## Options

Compare viable choices and their trade-offs.

## Decision

State the chosen option and the reason it is acceptable.

## Migration

Describe rollout, data migration, CI gates, and docs updates.

## Rollback

Describe how to reverse or contain the change.
