## "Looks-green-isn't-exercised" cross-check

From the pre-existing `inv/03-frontend.md` and `diag/06-frontend-quality.md`:

- The 100 % coverage reported in `coverage/test-evidence.json` for `angular-profile`, `angular-sessions`, `angular-trash` reflects how **thin** those packages are, not how thoroughly they are tested. A single export-existence test against a stub component yields 100 % line coverage of that stub.
- Conversely, the 4-scenario Playwright suite under `reference/web/test/e2e/` only exercises a tiny sliver of the matrix above (login + sample CRUD + trash-restore + flow auth-gate). Rows 4–6, 8, 17, 18, 22, 27 of this matrix are completely absent from Playwright. The "completeness" verdicts above were drawn from source inspection, not test execution — which means rows marked ✅ are tested at the unit level, rows marked 🟡 / 🟠 / 🔴 are not.

The rest of the FE-* files unpack each verdict.
