# STYNX Workspace

## Unreleased

- R16 deepens every published package README from ~50-130 line stubs to
  template-conformant developer references. All 41 packages (24 backend
  `@stynx/*`, 13 web `@stynx-web/*`, 4 tools `@stynx-internal/*`) now carry
  the locked 8-section shape: purpose / audience / install / quick-start /
  public-API-surface / configuration / examples / common-pitfalls /
  related-packages, pitched at the family-specific persona (NestJS backend
  devs / Angular frontend devs / workspace integrators). `@stynx/backend`
  (10 mountable submodules) and `@stynx/flow` (20 controllers / ~113 routes)
  split into `packages/<pkg>/docs/` subtrees; `sync-content.mjs` mirrors them
  into the published site. Two new round-gate scripts land under `scripts/`:
  `check-package-doc-shape.mjs` (asserts the 8 mandatory sections) and
  `verify-package-doc-coverage.mjs` (diffs README symbol cites vs index
  exports). `check-package-doc-shape` goes 0/41 → 41/41 clean;
  `check-docs-governance` holds at pass 14/14; the Docusaurus build is clean
  for every new cross-reference. Ratified by the round artifacts at
  `align/stynx/round-16/`; pilot retro at
  `docs/adopters/pilots/c-4/r16-package-deepening-retro.md` documents the
  devai `docs-links` sensor false-positive on Docusaurus site-absolute URLs
  (G16-1) and the barrel-re-export doc-coverage noise (G16-2).

- R15 migrates STYNX from DEVAI Constitution 0.1.1 to 0.2.0.
  `.devai/config/project.json` now carries an explicit
  `constitution.version: "0.2.0"` field. The `docs/` tree is reshaped
  from the 0.1.1 flat layout to the canonical seven-section IA
  (Start / Theory / Framework / Roles / Adopters / Reference / Meta) per
  DEVAI ADR-DOCS-IA. 17 directory `git mv` operations + 4 loose-root
  files kebab-cased on move (`Overview.md → start/index.md`,
  `Architecture Guide.md → framework/architecture-guide.md`,
  `RBAC Matrix.md → framework/rbac-matrix.md`,
  `KNOWN_GAPS.md → meta/known-gaps.md`); the previously top-level
  `docs/stynx/` directory (porting pack + status docs + working notes,
  76 files) merges under `docs/adopters/stynx/`. The new manifest
  `docs/_ia/categories.json` declares per-section labels + order +
  root-file allowlist per ADR-DOCS-IA. `docs/site/sidebars.js` is
  hand-authored to the seven sections (a mix of `type: 'doc'` for
  sections with only an index and `type: 'category'` with autogenerated
  items for sections with subdirectories). `docs/site/versioned_docs/`
  now carries both the 0.1.1 snapshot (139 files; the pre-R15 OLD
  layout) and the 0.2.0 snapshot (1951 files; the current layout) per
  the five-step recovery procedure documented at DEVAI R14 W05 commit
  `b700657`. `docs/site/versions.json` reads `["0.2.0", "0.1.1"]`.
  Five DEVAI dashboard generators (`gen-aspect-grid`,
  `gen-schema-browser`, `gen-self-scorecard`, `gen-test-matrix`,
  `gen-skill-catalog`) adopted as drop-in copies under `scripts/`;
  their outputs land at `docs/framework/aspect-grid.md`,
  `docs/framework/schemas/`, `docs/meta/{self-scorecard,test-matrix}.md`,
  `docs/reference/skills/`. `check-docs-governance` runs at 14 rules
  (9 R13 + 5 R14 IA) and returns verdict `pass`; the five new
  `docs-ia.*` rules all pass against STYNX's post-R15 state.
  `devai doctor --adopter` reports `f1-paths-present` ok for all
  seven enumerated 0.2.0 F1 paths. `docs publish --dry-run` validates
  the publishable artefact (8227 files / 161 MB) targeted at
  `gh-pages`. Adoption ratified by `docs/meta/adr/ADR-DEVAI-0.2.0-MIGRATION.md`;
  pilot retro at `docs/adopters/pilots/c-4/r15-migration-retro.md`
  documents 8 upstream-DEVAI gaps surfaced (G-1 through G-8) — most
  notable: `r14-*.mjs` scripts need `.pnpm-store/` + `.generated/`
  EXCLUDE_DIRS; generators need `--framework-base-url` for adopter
  portability; pack-override severity surface (Decision D) is not yet
  implemented upstream. R15 closes `with-blockers` because live
  `docs publish` to `gh-pages` and `pnpm ci:stynx` (Docker-dependent)
  are deferred for environmental reasons; round-scoped deliverables
  all landed clean.
